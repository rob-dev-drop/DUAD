REST JSON Flexible (Server side) Baja - Es el mas facil de iniciar y comprender Alto
SOAP XML Rigido Alto - Debido a su burocracia al necesitar un archivo WSDL y que sus peticiones sean exactas, tiene a querer ser implementado perfectamente todo el tiempo. Bajo, pero aun se usa en bancos y sistemas legacy
GraphQL JSON + lenguaje de consultas custom Flexible (Client side) Media - A la hora de empezar a utilizarlo es un poco mas dificil, pero escala mejor y esta ganando mas popularidad conforme pasa el tiempo. Medio

¿Cuál considera más apropiada para una startup moderna que desarrolla un sistema de reservas en línea?

Respuesta: REST. De entrada es el mas facil de implementar tanto en frontend como en backend. A pesar de los retos que implica usar este tipo de API, para un startup es lo mas optimo puesto que se puede implementar sin grandes inversiones de tiempo en configuracion y provee lo necesario para un buen funcionamiento y mantenimiento del sistema, que mientras se sigan buenas practicas de programacion, no deberia suponer un gran reto. A la hora de escalar, una migracion a GraphQL puede ser considerada si los beneficios lo ameritan.
